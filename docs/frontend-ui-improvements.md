# Що покращити у фронтенді та UI

## 1) Навігація та інформаційна архітектура (High)
- У застосунку змішані формати hash-роутингу: у ФІНІК використовується `#/page`, а у ФІЗРУК — `#page`. Це ускладнює deep-link, повернення назад та єдині патерни поведінки.
- Рекомендація: уніфікувати формат роутів (або перейти на React Router), додати централізовану карту маршрутів і явну обробку 404-станів.

## 2) Доступність (a11y) (High)
- Частина інтерактивних елементів має `div` з `onClick` замість семантичних кнопок (наприклад, рядок транзакції), що погіршує доступність з клавіатури та для screen reader.
- У базових UI-компонентів фокус-стан мінімальний або відсутній (`focus-visible` стилі для кнопок/інпутів не уніфіковані).
- Рекомендація: використовувати лише `button/a` для клікабельних елементів, додати `focus-visible` кільця, перевірити контраст тексту `text-subtle` на світлих фонах.

## 3) Узгодженість дизайн-системи (Medium)
- Багато стилів та SVG-іконок дублюються напряму в сторінках, що підвищує вартість підтримки.
- Рекомендація: винести AppHeader, BottomNav, Icon, PageContainer у спільні компоненти; зафіксувати токени (spacing/radius/shadow/typography) у `shared`.

## 4) Стани екранів та UX-мікровзаємодії (Medium)
- Лоадер є, але empty/error/offline стани реалізовані нерівномірно між модулями.
- Рекомендація: уніфікувати системні стани (loading/empty/error/success), додати retry-дiї та дружні пояснення.

## 5) Форматування даних і локалізація (Medium)
- Форматування дат/сум є локально у компонентах, але нема єдиного шару форматерів для всього UI.
- Рекомендація: централізувати форматери дати/валюти/чисел і використовувати один locale (`uk-UA`) у спільних утилітах.

## 6) Продуктивність та підтримуваність (Medium)
- Є lazy-loading сторінок — це плюс. Але великі компоненти (наприклад, транзакційний рядок) містять багато відповідальностей в одному файлі.
- Рекомендація: декомпозувати складні компоненти (row actions, split editor, category picker), додати memoization там, де є масивні списки.

## 7) Ясність сценаріїв для мобільного (Low/Medium)
- У проєкті якісно враховані safe-area insets, але UX жестів (свайп вкладок) не має явної індикації/налаштувань.
- Рекомендація: додати hint для жестів або опцію вимкнення свайпу, щоб уникнути випадкового перемикання.

## Перші кроки (на 1-2 спринти)
1. Уніфікувати роутинг між модулями + додати єдиний route config.
2. Вирішити a11y-борг: semantic buttons, `focus-visible`, keyboard-navigation.
3. Виділити shared layout primitives (`AppHeader`, `BottomNav`, `PageShell`).
4. Стандартизувати empty/error/loading стани у спільних компонентах.
