# Дорожня карта Hub (“Сержант”)

Ціль: тримати короткий, практичний список пріоритетів по модулях і спільному ядру Hub.

## Пріоритети (найближчі)

### Hub (ядро)
- **Консистентність структури/імпортів**: один стиль аліасів, без дублювань шляхів/файлів.
- **Стандарти UX**: спільні компоненти для `loading / empty / error`, однакові банери статусів.
- **Надійність storage**: версіоновані ключі, міграції, повідомлення при переповненні.

### ФІНІК (фінанси)
- **Стабілізація синку/імпорту** (Monobank): помилки, ретраї, прозорий статус синхронізації.
- **Аналітика**: категорії, бюджети, чиста вартість — довести до “щоденного використання”.

### ФІЗРУК (спорт)
- **План + прогрес**: узгодити структуру тренувань, прогресію, відновлення.
- **Швидкий старт тренування**: шаблони → активне тренування за 1–2 кліки.

### Рутина
- **Hub-календар як “шина”**: відображення ключових подій з Фініка/Фізрука в одному місці.
- **Ремайндери**: стабільність (дозволи, часові пояси), контроль частоти.

### Харчування
- **Фото → лог**: зменшити фрикцію збереження, краще відновлення стану після помилок API.
- **Локальна база продуктів**: імпорт/експорт, штрихкоди, оптимізація розміру storage.

## Беклог (пізніше)
- **Акаунти/синхронізація між пристроями** (мінімальна auth + серверне сховище).
- **Спільний пошук по Hub** (їжа/тренування/транзакції/звички).
