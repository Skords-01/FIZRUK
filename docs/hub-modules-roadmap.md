# Hub: модулі на оку (рішення та ТЗ)

Цей документ фіксує **пріоритет доменів** для розвитку Hub і **мінімальне ТЗ** для модуля «Звички», узгоджені з поточною архітектурою (`lazy`-модулі, `localStorage`, хаб у [`src/core/App.jsx`](../src/core/App.jsx)).

---

## 1. Пріоритет доменів (вибір)

| Пріоритет | Домен | Обґрунтування |
|-----------|--------|----------------|
| **1** | **Здоров’я / рутина** | Уже є сильний стовп **Фізрук** (тренування, заміри, календар плану); логічно нарощувати звички, сон, легке харчування без роздування Фініка. |
| **2** | **Продуктивність** | Нотатки, чеклісти, нагадування «крос-модульно» — після того, як дані користувача стабільно живуть в одному клієнті. |
| **3** | **Інфраструктура** | Синхронізація між пристроями, обліковий запис, service worker для push — **не** окремі «вкладки хаба», а шар під Фініком/Фізруком/майбутніми модулями. |

**Висновок:** на найближчий горизонт розробки варто тримати в фокусі **здоров’я та рутину** (нові модулі або розширення Фізрука), а інфраструктуру планувати паралельно лише якщо з’являється реальна потреба в multi-device без ручного JSON.

## 2. Модуль «Звички» (Habits) — мінімальне ТЗ

### 2.1 Мета

Простий трекер щоденних звичок: відмітка виконання по днях, **streak**, без сервера.

### 2.2 Дані (чернетка схеми)

Зберігання в `localStorage`, один ключ на сховище, наприклад `hub_habits_v1`:

```json
{
  "schemaVersion": 1,
  "habits": [
    {
      "id": "uuid",
      "name": "Вода 2 л",
      "emoji": "💧",
      "createdAt": "ISO-8601",
      "archived": false
    }
  ],
  "completions": {
    "uuid": ["2026-04-13", "2026-04-12"]
  }
}
```

- Дата — рядок **`YYYY-MM-DD`** у локальному календарі користувача.
- Ідентифікатор звички — стабільний `id`; `completions[id]` — відсортований або ні масив дат (дедуплікат при записі).

### 2.3 UI (мінімум)

- Список активних звичок + кнопка «Додати».
- На головному екрані модуля: **сітка поточного місяця** або горизонтальний ряд останніх 7–14 днів (як у Фізруку за складністю).
- Тап по дню — toggle виконання для обраної звички (або long-press для вибору звички, якщо одна картка на день).

### 2.4 Інтеграція з Hub

- Новий lazy-модуль: `src/modules/habits/HabitsApp.jsx` (або `HubHabits.jsx`).
- У [`App.jsx`](../src/core/App.jsx): додати `habits` до `VALID_MODULES` і об’єкта вибору модуля (іконка + опис).
- Експорт у повний бекап Hub (за аналогією з [`fizrukStorage`](../src/modules/fizruk/lib/fizrukStorage.js) / окремий `hubBackupKeys` у корені, якщо з’явиться спільний експорт).

### 2.5 Не входить у v0

- Спільний акаунт / синхронізація.
- Push-нагадування (можна пізніше через той самий SW, що й для Фізрука).
- Складна аналітика (лише streak + календар відміток).

### 2.6 Зв’язок з Фізруком (опційно пізніше)

- Діплінк з дашборду Фізрука: «Звички» як доповнення до тренувань, без обов’язкового merge даних у v0.
